  • Thoughts on Progressive Enhancement

    After reading the article, i have the temptation to write off progressive enhancement as a relic of the browser wars. I mean, who turns off JS today? Is that even possible (it is)? Auto-updates have made outdated browsers a thing of the past (they're all chromium anyway lol). However, i'd argue that the ideas in the article are more important today in the age of the overengineered web. We have a million options available to us now to complicate webpages. HTML5 alone can do things that were only available through plugins in times past. CSS can do what required JS before. On top of that, there are frameworks available for every use case. I understand progressive enhancement as taking care to build semantic webpages, utilizing the proper features of each persona of the "web trinity" before adding complexity, i.e. build a semantic navigation bar using the HTML5 nav element wrapped in a header element, with any animations or collapsing behaviors added via CSS beginning with the mobile layout in mind, finally bringing in JS only where absolutely necessary for dynamic interactions. This will result in cleaner markup and a more consistent experience across devices.
